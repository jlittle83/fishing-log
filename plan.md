1. Project Overview & Scope
Objective
Develop a responsive web application that enables anglers to:

Log fishing catches with detailed metadata
Track personal performance over time
Visualize trends and statistics through dashboards

Core Features (In Scope)

Catch logging (CRUD operations)
Data visualization (charts, trends, summaries)
Personal best tracking
Filtering & search (by species, date)
Responsive UI (desktop + mobile)

Out of Scope (MVP)

User authentication / accounts
Cloud sync or multi-device support
Social sharing / community features
Real-time collaboration
Marketplace or gear tracking
Weather API integration
Photo attachments
Map-based location entry
Export to CSV


2. Target Users
Primary Users

Recreational anglers tracking personal performance
Competitive anglers analyzing catch trends

Secondary Users

Fishing enthusiasts interested in data insights
Tournament fishers needing quick stats

User Personas
Casual Casey — Fishes on weekends, wants to log catches quickly, minimal friction
Data-Driven Dana — Fishes frequently, wants trends, charts, and filtering
Tournament Terry — Needs fast access to personal bests and location history
(Full persona details in requirements.md)

3. Assumptions

Single user per device (no multi-user support needed)
Users have internet access and a modern browser
localStorage provides sufficient capacity for typical angler data
Users will enter data manually (no GPS or sensor input)
No account creation required for MVP


4. Constraints
Technical

Frontend only — no backend server for MVP
Must be buildable as a single React application
No paid APIs or services
localStorage as sole data persistence layer

Resource

Solo developer
No design budget — rely on Tailwind CSS

Time

MVP target: 2–3 weeks
